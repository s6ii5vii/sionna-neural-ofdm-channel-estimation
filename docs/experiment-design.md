# Experiment design

## System overview

The implemented baseline uses independent, unit-power Rayleigh channel
coefficients and a pilot observation model:

```text
y = h*x + n
```

where `h` is the complex channel coefficient, `x` is a known non-zero pilot,
and `n` is circular complex Gaussian noise. This model is a controlled starting
point, not a complete OFDM physical layer.

## OFDM setup

The current experiments use 64 coefficient locations as a simplified
subcarrier/pilot axis. Earlier notebooks explored a 64-subcarrier, 14-symbol
resource grid with a cyclic prefix. A future Sionna implementation must promote
those exploratory settings into a validated, configuration-driven transmitter,
channel, and receiver pipeline.

## Pilot-based estimation

Unit-valued complex pilots are generated by default. Pilot density can select a
uniform subset of locations. Sparse-pilot LS results are valid only at observed
locations until interpolation across the full grid is implemented.

## LS baseline

For non-zero pilots, the implemented estimator computes:

```text
h_hat_LS = y / x
```

This is the primary reproducible baseline. An LMMSE function is present only as
an explicit `NotImplementedError`; correct covariance assumptions must be
specified before implementation.

## Planned neural estimator

The initial neural skeleton consumes real/imaginary pilot features and predicts
real/imaginary channel features. It uses one configurable hidden dense layer.
Training results are not yet part of the repository and must be compared against
LS using identical test conditions.

## Dataset generation

Synthetic datasets are generated from seeded independent Rayleigh coefficients,
unit pilots, and AWGN. Complex arrays are represented with a final axis of
length two. Splits are deterministic and currently preserve generation order.
Future versions should define SNR sampling, shuffling, channel-model variation,
and artifact metadata in configuration.

## Evaluation metrics

- NMSE measures channel-estimation error relative to reference channel power.
- BER is available for future end-to-end symbol or bit decisions.
- Planned efficiency metrics include parameter count, model size, latency, and
  training-data volume.

Metrics must be reported per experiment profile with uncertainty or repeated
seeds where appropriate.

## Experiment profiles

`baseline-ls` evaluates the implemented LS estimator across 0-30 dB SNR with all
64 pilot locations.

`low-resource-v1` defines an initial constrained profile with lower SNR values,
25% pilot density, fewer evaluation samples, a training-dataset target, and a
model-parameter target. Its current runner evaluates only the implemented LS
portion.

## Reproducibility plan

1. Keep all reusable logic in `src/channel_estimation/`.
2. Store experiment settings in versioned YAML files.
3. Record random seeds and output paths.
4. Generate figures and CSV tables from experiment runners.
5. Test small known cases independently of TensorFlow and Sionna.
6. Record environment details for any reported result.
7. Avoid manual edits to generated result tables.
8. Add repeated-seed evaluation before drawing scientific conclusions.
