# Experiment design

## System overview

The implemented baseline uses independent, unit-power Rayleigh channel
coefficients and a pilot observation model:

```text
y = h*x + n
```

where `h` is the complex channel coefficient, `x` is a known non-zero pilot,
and `n` is circular complex Gaussian noise. This model is a controlled starting
point, not a complete OFDM physical layer.

## OFDM setup

The simplified experiments use 64 coefficient locations as a pilot axis. The
Sionna experiment uses a configured 72-subcarrier, 14-symbol resource grid with
a cyclic prefix, pilot-bearing symbols, random QAM payload data, a 3GPP TDL
channel, and additive white Gaussian noise.

## Pilot-based estimation

Unit-valued complex pilots are generated by default. Pilot density can select a
uniform subset of locations. The Sionna grid evaluates nearest-neighbor, linear,
and covariance-informed LMMSE interpolation across the full grid.

## LS baseline

For non-zero pilots, the implemented estimator computes:

```text
h_hat_LS = y / x
```

This is the primary simple baseline. A NumPy LMMSE implementation uses an
explicit estimated channel covariance, while the Sionna grid comparison uses
TDL-derived time and frequency covariance matrices. The latter is reported as a
model-informed benchmark because it receives the configured channel profile.

## Neural estimator

The grid CNN consumes an LS-lin full-grid estimate represented by real and
imaginary features and predicts a refined full-grid channel. Training uses an
independent data seed, a model seed, validation-based checkpoint selection, and
an independent evaluation seed. Results must be compared with all classical
baselines under identical evaluation conditions.

## Dataset generation

Synthetic datasets are generated from seeded channel realizations, known pilots,
and AWGN. Complex arrays are represented with a final axis of length two. Splits
are deterministic and preserve generation order. Grid datasets include a
configuration fingerprint so incompatible cached data is regenerated. Future
versions should add mixed-SNR training and richer artifact storage.

## Evaluation metrics

- NMSE measures channel-estimation error relative to reference channel power.
- BER is available for future end-to-end symbol or bit decisions.
- Planned efficiency metrics include parameter count, model size, latency, and
  training-data volume.

Metrics must be reported per experiment profile with uncertainty or repeated
seeds where appropriate.

## Experiment profiles

`baseline-ls` evaluates the implemented LS estimator across 0-30 dB SNR with all
64 pilot locations.

`low-resource-v1` defines an initial constrained profile with lower SNR values,
25% pilot density, fewer evaluation samples, a training-dataset target, and a
model-parameter target. Its current runner evaluates only the implemented LS
portion.

## Reproducibility plan

1. Keep all reusable logic in `src/channel_estimation/`.
2. Store experiment settings in versioned YAML files.
3. Record random seeds and output paths.
4. Generate figures and CSV tables from experiment runners.
5. Test small known cases independently of PyTorch and Sionna.
6. Record environment details for any reported result.
7. Avoid manual edits to generated result tables.
8. Use repeated-seed validation before drawing scientific conclusions.
